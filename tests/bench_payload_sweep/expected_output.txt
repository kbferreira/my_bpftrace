Expected JSON schema and example output for bench_payload_sweep.py
==================================================================

JSON Schema (keys and types)
-----------------------------
{
  "benchmark":        string  -- always "bench_payload_sweep"
  "transport_results": {
    "BPF_PERF_OUTPUT": [
      {
        "payload_bytes":   integer -- size of the event payload in bytes
        "count":           integer -- number of valid latency samples collected
        "mean_ns":         float   -- arithmetic mean end-to-end latency (nanoseconds)
        "p50_ns":          float   -- 50th-percentile latency (nanoseconds)
        "p95_ns":          float   -- 95th-percentile latency (nanoseconds)
        "p99_ns":          float   -- 99th-percentile latency (nanoseconds)
        "stdev_ns":        float   -- population standard deviation (nanoseconds)
        "min_ns":          float   -- minimum observed latency (nanoseconds)
        "max_ns":          float   -- maximum observed latency (nanoseconds)
        "events_per_sec":  float   -- throughput: events delivered per second
        "elapsed_sec":     float   -- total wall-clock time for this data point (seconds)
      },
      ...  (one entry per payload size)
    ],
    "BPF_RINGBUF": [
      -- same structure as BPF_PERF_OUTPUT, or:
      {
        "payload_bytes": integer -- size of the event payload in bytes
        "skipped":       true    -- present on kernels < 5.8
        "reason":        string  -- human-readable skip reason
      }
    ]
  }
}

Notes
-----
- Latency is measured from bpf_ktime_get_ns() in the kernel probe to
  time.monotonic_ns() at the start of the Python callback (same methodology
  as bench_perf_output.py and bench_ringbuf.py).
- Expected trend: latency and elapsed_sec increase with payload_bytes because
  the BPF-to-userspace copy path must transfer more data per event.
- BPF_RINGBUF typically shows lower latency than BPF_PERF_OUTPUT at the same
  payload size due to its zero-copy ring buffer design.
- On kernels < 5.8, all BPF_RINGBUF entries will contain "skipped": true
  instead of latency metrics.
- The inflection point where latency growth accelerates (usually around
  512–1024 bytes) corresponds to L1/L2 cache line boundary effects.
